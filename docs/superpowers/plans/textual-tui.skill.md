# Textual TUI for Mindfulness Neurofeedback

## Design Intent

- **Statement**: Given Python 3.13+ with Textual 8.x, Apptainer-managed MURFI, FSL, and pynetdicom on a single Ubuntu workstation, applying FCIS architecture with a Textual TUI produces a foolproof operator interface that prevents silent data loss via traffic light validation and single-keypress operation.
- **Approach (A)**: Replace all bash scripts with a Python package (`mindfulness_nf/`) split into functional core (frozen models, pure quality functions) and imperative shell (async orchestration + Textual TUI).
- **Quality Outcome (Q)**: Non-technical RAs operate the full neurofeedback pipeline without ambiguous choices. MoCo misconfiguration is caught before data loss. Volume count validation blocks advancement on insufficient data.
- **Rationale (R)**: FCIS isolates testable logic from I/O. Textual requires no installation beyond pip. Traffic light model replaces retry/skip/quit decisions with green/yellow/red status.
- **Constraints (K)**:
  - Python 3.13.12 at `/usr/local/bin/python3.13` (verified)
  - Textual 8.1.1 available on PyPI (verified)
  - MURFI container at `/opt/murfi/apptainer-images/murfi.sif` (verified, 511MB)
  - Apptainer 1.3.4 (verified)
  - FSL at `/opt/fsl` (verified)
  - pynetdicom 3.0.4+ for DICOM receiver (verified in pyproject.toml)
  - Scanner at 192.168.2.1, vSend port 50000, DICOM port 4006 (verified)
  - PsychoPy run duration is 150 seconds (verified, not 900s)
  - Scale factor does NOT reset at Run 8 (verified)
  - `onlyReadMoCo` only in `2vol.xml` and `rtdmn.xml` (verified)
  - `.venv` currently uses Python 3.12.13; must be recreated with 3.13

## Invariants

1. **FCIS boundary**: No I/O imports (`asyncio`, `subprocess`, `socket`, `pathlib.Path.read_*`, `open()`) in `models.py`, `quality.py`, or `config.py`. These files contain only frozen dataclasses, enums, and pure functions.
2. **Immutability**: All domain models use `frozen=True, slots=True`. Collections are `tuple`, not `list`. Updates return new instances via `copy.replace()`.
3. **Type safety**: All public functions have type annotations. Use `X | None` (not `Optional[X]`), `list[T]` (not `List[T]`), `match`/`case` with `assert_never` for exhaustive handling.
4. **Async correctness**: Use `asyncio.TaskGroup` (not `gather`). Use `asyncio.to_thread()` for blocking subprocess calls. Always re-raise `CancelledError` after cleanup.
5. **Test purity**: Core tests use NO mocks. Shell tests mock only external I/O (subprocesses, network). Never mock internal code.
6. **No mutable defaults**: Use `None` sentinel + factory pattern.
7. **PsychoPy untouched**: `psychopy/balltask/` is read-only. The orchestration library launches it as a subprocess.
8. **Existing data preserved**: `murfi/subjects/` and `murfi/scripts/` are not modified or deleted.
